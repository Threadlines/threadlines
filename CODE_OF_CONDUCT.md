# Code of Conduct

## Our Standard

Threadlines should be a constructive place to build software. Be respectful,
assume good intent, critique ideas and code rather than people, and make room for
contributors with different backgrounds and experience levels.

Harassment, discrimination, threats, personal attacks, doxxing, and sustained
disruptive behavior are not acceptable in project spaces.

## Project Spaces

This policy applies to the repository, issues, pull requests, discussions,
release channels, and other spaces represented as part of Threadlines.

## Reporting

For ordinary moderation concerns, use GitHub's report and block tools. For a
private report involving a maintainer or sensitive personal information, open a
draft [GitHub Security Advisory](https://github.com/Threadlines/threadlines/security/advisories/new)
and begin the title with `Conduct report:`. Do not include credentials, private
repository contents, or unrelated personal data.

Maintainers may remove content, limit participation, or ban contributors when
needed to protect the project and its community.
