Threadlines/Threadlines multi-panel sidebar ideas

Recommendation

  1. Browser context capture first. Open a localhost URL, select an element or region, annotate it, and attach it to the composer.
  2. Use a temporary focus surface. Browser previews need width. Keep source control anchored as a compact repo strip while the preview is active.
  3. Do not move terminal into the right panel yet. The bottom drawer is still a better shell surface than another permanent column.
  4. Treat file browsing as context. File preview should help attach/read context, not become a full in-app editor.
  5. Defer full browser automation. Start with snapshot + screenshot + selected context.
