Setting up the hero screenshot

How to capture the real "app working" shot.

Both reference sites sell with one strong screenshot of the product mid-task. The mockups above fake that frame so you can judge the layout. To shoot the real one, stage the app like this.

1 What to show

  • All three columns visible at once: threads rail, chat, and the source control panel. The panel is the differentiator, so keep it open.
  • One thread actively running so the live node and a "Working" row are on screen.
  • A real diff in the panel: a few changed files with per-file +/- counts and a branch name like thread/refine-scp.
  • A recognizable model in the composer (Claude Opus 4.8 or a Codex model) so providers read instantly.

2 Staging

  • Use a clean demo repo with believable thread titles. Avoid private paths and real tokens.
  • Mid-turn beats finished: an agent partway through editing tells the story better than an idle screen.
  • Keep the prose short and legible at the size it renders. Long messages turn to grey mush when scaled down.
  • Dark theme for Thread and Spine; for Workbench either dark (high contrast on the light page) or a light capture if you want to prove light mode.

3 Capture

  • Shoot at 2x for retina. Target a 16:10-ish window around 1440×900 or 1600×1000.
  • Headless Edge works well here: --force-device-scale-factor=1 on scaled displays, per the brand-kit recipe.
  • Export a transparent or trimmed PNG, then let the page supply the window frame and shadow.
  • Hide OS chrome and personal data. The faux titlebar in these mockups can stand in for the real one.

4 Swapping it in

  • Each mockup's app window is one .shot-frame / .appwin block. Replace its inner markup with an <img> of the capture.
  • Keep the rounded frame, border, and shadow. They do the heavy lifting and stay consistent across directions.
  • Consider a short looping clip later: a thread finishing, the diff updating, the commit landing on the graph.
  • Whatever you pick moves into apps/marketing/ as the real implementation. Nothing here touches it yet.
