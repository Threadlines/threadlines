The source control panel

A real review surface, not a footnote.

Most agent tools bury git behind a chat command. Threadlines puts a live source control panel beside every conversation, so you watch the working tree change as the agent edits, then stage, commit, and open a pull request in place.

  • Per-file diff stats grouped by status, polled live every few seconds and on focus.
  • Commit, push, and open a PR from one panel, with a branch per thread.
  • GitHub, GitLab, Azure DevOps, and Bitbucket for pull and change requests, all in the same place.

Parallel threads

Run many agents. Keep every line straight.

Start a thread per task and let Claude and Codex work side by side, each in its own isolated branch. The thread graph keeps the picture readable, so a glance tells you what is running, what is done, and what needs you.

  • One live node per surface. The accent only ever means now, running, current.
  • Amber means needs you. Approvals and questions surface without stealing focus.
  • Switch providers per thread. Pick the model that fits the task, keep the branch.

Built for the whole loop

Everything around the conversation.

Built-in terminal and live diffs

An xterm terminal and rich, changes-only diffs sit inside the workspace. Read exactly what an agent touched without tab-switching to your editor or a browser.

Permission modes that fit

From full-access to a guarded auto tier with review, set the trust level per provider.

Resilient under load

Reconnects, session restarts, and partial streams are handled so a flaky network never loses your place in a turn.

Local and desktop-native

Runs on your machine against your repos and your keys. Nothing routes through us.

Open source

Read it, fork it, ship your own. A friendly fork of t3 with deeper source control.
