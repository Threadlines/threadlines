threadlines / design explorations

Six directions

Each frame below is the same Tuesday: the Diff Viewer thread is mid-run, three files are dirty, one commit just landed. Same projects, same threads, same commits as v0.0.20, so the directions compare honestly. Loom and Ledger carry over from round one; Chorus, Tandem and Filament are new, each a system idea native to orchestrating agents rather than a borrowed style, and Spool is Ledger turned chat-first. Everything is dark first, and every frame flips to a tuned light mode with the toggle. The usual rules hold: one live node per surface, the palette opens with Search, no hotkey chips on content.

WHAT CHANGES

The conversation gains a thread rail. Prompts, agent work, commits and the live run are nodes on one line, and the commit graph in the panel continues that same line. Messages lose their boxes; the rail carries the structure instead.

WHY IT GETS EASIER

One scan answers the three questions that matter: what happened, what got committed, what is still running. Chat history and git history stop being two separate stories you reconcile in your head.

WATCH OUT

The rail only stays calm with strict event grouping. If every tool call becomes a node, the line turns into noise. Collapse agent work into one node per turn and let the diff panel hold the detail.

WHAT CHANGES

The workspace holds two or three live threads side by side. A conductor rail across the top carries one bead per run on a single connecting line, including runs that are not open as columns. States are choreographed: the focused column carries the blue ring, running columns dim slightly, and the one waiting on you raises an amber edge.

WHY IT GETS EASIER

This is orchestration without a dashboard. You steer parallel agents in one room instead of hopping tabs, approvals stop blocking silently, and the rail answers what is alive from anywhere. The empty state stays the home you already like; Chorus only assembles when a second thread goes live.

WATCH OUT

Columns need a width floor, and past three they should collapse into rail beads. The focus ring has to be unmistakable so a prompt never lands in the wrong thread, and slim composers need a clear expand affordance.

WHAT CHANGES

The thread splits into two synchronized lanes. Conversation prose stays on the left, every action the agent takes streams on the right, and a center spine ties each reply to the work it produced. The composer lives under the talk lane; commit controls live under the work lane.

WHY IT GETS EASIER

Long agent turns stop burying the conversation. You read intent in seconds, audit actions only when you care, and pointing at either side lights up its counterpart, shown on the middle rung. Tool output finally has a home that is not your chat history.

WATCH OUT

Grouping rules must stay honest, one work group per turn, or the ties tangle. Both lanes have to scroll as a single surface, and narrow windows need a collapse where the work lane folds back into inline cards.

WHAT CHANGES

The sidebar is gone. Navigation is the threadline figure itself: each project is a trunk, each thread branches off it, and state lives on the line. The live branch glows and its node pulses, a waiting project carries an amber dot, merged threads get tied off, and the active branch flows out of the nav into the thread header.

WHY IT GETS EASIER

WATCH OUT

Graphs scale worse than lists. Past eight or so threads a trunk must collapse to recents plus a count, with the full list one click away, and labels always stay horizontal. The glow is reserved for the single live node so the nav never turns into a light show.

WHAT CHANGES

The workspace inverts. A horizontal threadline pins time across the top with the live amber node at its leading edge, the diff working set takes center stage with inline previews, chat docks left as a steering column, and commit controls sit beside the files they act on.

WHY IT GETS EASIER

When the agent does the typing, your real job is judging the work. Review and commit happen where your eyes already are, chat never buries the diff, and the timeline permanently answers where you are: what landed, what is uncommitted, what comes next.

WATCH OUT

Chat loses width, so long agent explanations need collapsing summaries. This works best as a mode the app enters when changes exist, paired with a chat-first layout like Loom for the conversation-heavy start of a thread.

Same bones as Ledger, different center of gravity. The timeline stays on top, but the conversation returns to the middle at full width. WATCH OUT

Built from the live app: same projects, threads and commits as v0.0.20. Frames are static HTML at 1380×860, scaled to fit, each with a tuned light palette behind the toggle. Loom is the nearest step from today. The six compose rather than compete: Loom is the thread at rest, Tandem is the same thread under audit, Ledger is its review mode and Spool is that mode at ease, Chorus is several threads at once, and Filament is how you move between them.
